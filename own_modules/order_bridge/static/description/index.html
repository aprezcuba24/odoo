Tienda Apk

API REST para clientes externos: registro de dispositivos, catálogo de productos y pedidos de venta.

Expone puntos finales JSON bajo /api/order_bridge/ para apps móviles, quioscos u integraciones. Los clientes se autentican con una clave de dispositivo (token Bearer), no con usuarios de Odoo.

Qué debe configurar

  1. Productos del catálogo — en cada plantilla de producto, active Visible en Tienda Apk (junto a las mismas opciones que Se puede vender). Solo los productos activos, vendibles y con esta marca (y que cumplan las reglas de empresa) aparecen en /categories y /products y pueden añadirse a pedidos por API.
  2. Validar dispositivos en el backend — tras POST /api/order_bridge/register con teléfono y clave de dispositivo, el personal abre Ventas → Tienda Apk → Dispositivos (o la pestaña Tienda Apk del contacto) y valida el teléfono cuando corresponda. Los pedidos pueden crearse antes de la validación; quedan marcados en el pedido de venta para revisión.

Dónde gestionar dispositivos en Odoo

Este módulo depende de la aplicación Ventas (sale): Ventas se instala automáticamente con Tienda Apk y debe permanecer instalada. Los menús de dispositivos, pedidos y clientes de la tienda están bajo Ventas: Ventas → Tienda Apk (Dispositivos, Pedidos Tienda Apk, Clientes Tienda Apk).

Solo los usuarios con acceso de comercial (o un grupo de seguridad que lo incluya) ven esos menús y la pestaña Tienda Apk en contactos. Desde un contacto puede usar Abrir dispositivos para filtrar la lista por ese cliente. Los pedidos de venta muestran el dispositivo API vinculado y la instantánea de validación en el formulario cuando aplica.

Las aplicaciones externas usan la API REST y las claves de dispositivo; no usan estos menús del backend.

Dependencias y vista general de la API

Requiere Ventas, Productos y Validación de teléfono (para números normalizados).

Rutas principales (todas bajo la URL base de Odoo):

  • POST /api/order_bridge/register — registrar dispositivo (sin autenticación)
  • GET /api/order_bridge/status — estado del dispositivo (Bearer)
  • GET /api/order_bridge/categories, GET /api/order_bridge/products — catálogo (Bearer)
  • GET|POST /api/order_bridge/orders, GET /api/order_bridge/orders/<id>, POST /api/order_bridge/orders/<id>/cancel — pedidos (Bearer)
  • GET|PUT|PATCH /api/order_bridge/profile — perfil (Bearer)
