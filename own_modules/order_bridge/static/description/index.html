Order Bridge

REST API for external clients: device registration, POS-aligned catalog, and sale orders.

Exposes JSON endpoints under /api/order_bridge/ for mobile apps, kiosks, or integrations. Clients authenticate with a device key (Bearer token), not Odoo users.

What you must configure

  1. Link a point of sale per company — required for catalog and order creation from the API. Go to Settings → Point of Sale, open the Order bridge block, and set Order bridge point of sale to a pos.config whose products and category rules should match what external clients see. Alternatively, set the same field on the company form (Order bridge tab).
  2. Without this link, catalog and order routes respond with 503 and an error indicating missing configuration.
  3. Validate devices in the backend — after a client calls POST /api/order_bridge/register with phone and device key, staff open Sales → Order bridge → Devices (or the contact's Order bridge tab) and validate the phone when appropriate. Orders are still allowed before validation; they are flagged on the sale order for review. See the next section for who can open these screens.

Where to manage devices in Odoo

This module depends on the Sales app (sale): Sales is installed automatically with Order Bridge and must stay installed. The backend menus for devices, bridge orders, and bridge customers are under the Sales app: Sales → Order bridge (Devices, Bridge orders, Bridge customers).

Only users with Salesperson access (or a security group that includes it) see those menus and the Order bridge tab on contact forms. From a contact you can also use Open devices to filter the device list to that customer. Sale orders show the linked API device and validation snapshot on the order form when relevant.

External apps use the REST API and device keys; they do not use these backend menus.

Dependencies and API overview

Requires Sales, Point of Sale, Products, and Phone validation (for normalized phone numbers).

Main routes (all under your Odoo base URL):

  • POST /api/order_bridge/register — register device (no auth)
  • GET /api/order_bridge/status — device state (Bearer)
  • GET /api/order_bridge/categories, GET /api/order_bridge/products — catalog (Bearer)
  • GET|POST /api/order_bridge/orders, GET /api/order_bridge/orders/<id>, POST /api/order_bridge/orders/<id>/cancel — orders (Bearer)
  • GET|PUT|PATCH /api/order_bridge/profile — profile (Bearer)
